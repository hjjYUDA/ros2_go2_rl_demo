# Go2 + IsaacLab 强化学习：实现说明与应用指南

本文档说明本项目中 **训练（IsaacLab / unitree_rl_lab）** 与 **部署（ROS 2 + MuJoCo 仿真）** 的职责划分、具体实现要点，以及如何在本机或另一台电脑上查看运行效果。

---

## 一、整体架构

```
训练侧（GPU，IsaacSim + IsaacLab）
  └─ 任务：Unitree-Go2-Velocity-Robust（增强域随机化）
  └─ 产出：model_XXXX.pt、params/deploy.yaml

导出（可选在无图形界面下完成）
  └─ export_onnx_standalone.py → policy.onnx

部署侧（ROS 2 Humble）
  └─ mujoco_sim_node.py    ：MuJoCo 物理 + /go2/joint_states、/go2/imu
  └─ rl_inference_node     ：读取 ONNX + deploy.yaml，PD 输出 /go2/cmd_torque
  └─ keyboard_teleop_node.py：键盘 → /go2/cmd_vel、/go2/mode
```

**关节顺序（所有 ROS 话题一致，Unitree SDK 顺序）：**

`FR_hip → FR_thigh → FR_calf → FL_* → RR_* → RL_*`（每条腿 hip/thigh/calf）。

策略内部使用 Isaac 资产顺序；`deploy.yaml` 里的 `joint_ids_map` 记录 **资产索引 → SDK 索引**，推理节点已按官方含义实现。

## 二、训练侧做了什么

| 项目 | 说明 |
|------|------|
| 任务注册 | `velocity_robust_env_cfg.py`：更强的推力、质量、摩擦与初始速度随机化 |
| 训练命令示例 | `python scripts/rsl_rl/train.py --task Unitree-Go2-Velocity-Robust --num_envs 4096 --max_iterations 5000 --headless` |
| 日志目录 | `unitree_rl_lab/logs/rsl_rl/unitree_go2_velocity_robust/<时间戳>/` |
| 关键产物 | `model_XXXX.pt`、`params/deploy.yaml`（观测缩放、PD、指令范围等） |

**查看训练是否还在跑：**

```bash
pgrep -af "scripts/rsl_rl/train.py"
tail -f /tmp/go2_train.log    # 若启动时重定向到此文件
nvidia-smi                     # 确认 GPU 占用
```

日志中的 **ETA** 为当前一次完整训练（到 `max_iterations`）的预计剩余时间。

## 三、部署侧做了什么（ROS 2 包 `go2_rl_control`）

| 节点 | 作用 |
|------|------|
| `rl_inference_node`（C++） | 加载 `policy.onnx` 与 `deploy.yaml`；按 `deploy.yaml` 拼观测（45 维）；50 Hz 推理；PD 控制生成力矩 |
| `mujoco_sim_node.py` | 加载 MuJoCo Menagerie 的 Go2 `scene.xml`；1 kHz 仿真；约 200 Hz 发布关节状态与 IMU；订阅 `/go2/cmd_torque` |
| `keyboard_teleop_node.py` | WASD/QE 增量速度；1/2/3 切换 stand / walk / passive |

**话题摘要：**

- 订阅：`/go2/joint_states`、`/go2/imu`、`/go2/cmd_vel`（`geometry_msgs/Twist`）、`/go2/mode`（`std_msgs/String`：`stand`/`walk`/`passive`）
- 发布：`/go2/cmd_torque`（`Float64MultiArray`，长度 12）
- 可选扰动：`/go2/disturb`（`geometry_msgs/Vector3`，冲量 N·s，作用于机体线动量）

## 四、编译与环境

**依赖（Ubuntu 22.04 + ROS 2 Humble）：**

```bash
sudo apt install ros-humble-rclcpp ros-humble-sensor-msgs ros-humble-geometry-msgs \
  ros-humble-launch-ros libyaml-cpp-dev libeigen3-dev
pip3 install --user mujoco   # MuJoCo Python，需与运行 ros2 的 Python 一致（建议系统 /usr/bin/python3）
```

**ONNX Runtime：** 默认 CMake 查找 `/opt/onnxruntime`（可按安装路径调整）。

**编译：**

```bash
cd ~/ros2_ws
colcon build --symlink-install --packages-select go2_rl_control
source install/setup.bash
```

**注意（重要）：** ROS 2 的 Python 节点必须与 **系统自带的 Python 3**（Ubuntu 22.04 上一般为 `/usr/bin/python3`）配合使用，这样 `rclpy` 自带的 C++ 扩展才能加载。

本仓库里 `mujoco_sim_node.py`、`keyboard_teleop_node.py` 的 shebang 已固定为 **`#!/usr/bin/python3`**，避免在 **Conda 环境已激活**（例如 `env_isaaclab`）时，`env python3` 指向 Conda、因 **`libstdc++.so.6` 过旧** 而出现：

`GLIBCXX_3.4.30 not found (required by ... _rclpy_pybind11 ...)`

若仍报错，请先 **`conda deactivate`**，再 `source /opt/ros/humble/setup.bash` 与 `source ~/ros2_ws/install/setup.bash`。训练 IsaacLab 时再单独 `conda activate env_isaaclab`。

## 五、训练结束后：导出策略并刷新 ROS 包

训练目录下会有最新 `model_XXXX.pt` 与 `params/deploy.yaml`。

**一键刷新（推荐）：**

```bash
chmod +x ~/ros2_ws/src/go2_rl_control/scripts/refresh_policy.sh
~/ros2_ws/src/go2_rl_control/scripts/refresh_policy.sh
```

脚本会：调用 `export_onnx_standalone.py` 导出 ONNX → 拷贝到 `go2_rl_control/policy/` → `colcon build`。

**手动指定某个 checkpoint：**

```bash
~/ros2_ws/src/go2_rl_control/scripts/refresh_policy.sh \
  /home/ubuntu/unitree_rl_lab/logs/rsl_rl/unitree_go2_velocity_robust/<run>/model_5000.pt
```

## 六、在本机运行演示

**终端 A — 仿真 + 推理（不要在 Conda 里跑；若提示符里有 `(env_isaaclab)` 请先 `conda deactivate`）：**

```bash
conda deactivate   # 避免 conda 的 Python/libstdc++ 导致 rclpy 加载失败
source /opt/ros/humble/setup.bash
source ~/ros2_ws/install/setup.bash

ros2 launch go2_rl_control sim.launch.py rl_mode:=stand visualize:=true
```

**终端 B — 键盘（需在真实 TTY 或带键盘焦点的终端）：**

```bash
conda deactivate
source ~/ros2_ws/install/setup.bash
ros2 run go2_rl_control keyboard_teleop_node.py
```

操作提示：`2` 进入行走模式，`W/S/A/D/Q/E` 调速度，`1` 站立，`3` 被动，`空格` 清零速度。

**无界面服务器：** 将 launch 中 `visualize:=false`，仅用话题观察：

```bash
ros2 topic hz /go2/joint_states
ros2 topic echo /go2/base_pose --once
```

### 六点五、`visualize:=true` 仍没有 MuJoCo 窗口？

MuJoCo 使用 **GLFW + OpenGL** 弹窗，**必须在有图形显示的环境下运行**。下面是最常见原因与处理：

| 情况 | 说明与处理 |
|------|------------|
| **纯 SSH、无 `-X`** | 当前 shell 没有 `DISPLAY`，窗口无法创建。先执行 `echo $DISPLAY`，若为空：在本机**图形桌面**里打开终端再运行；或 `ssh -X user@主机` / `ssh -Y` 后再 launch。 |
| **云 GPU / 无桌面** | 没有物理显示器时通常也没有可用的 X11。需要 **VNC / noVNC / 云厂商「远程桌面」** 装上桌面后再在该会话里运行 `ros2 launch`。 |
| **有桌面但只 SSH 登录** | 若机器上已有用户图形会话，可尝试在同一用户下 `export DISPLAY=:0`（或 `:1`，用 `echo $DISPLAY` 在桌面终端里看实际值），再启动 launch。**权限**上可能还需 `xhost +local:`（慎用，仅内网）。 |
| **Docker** | 需挂载 X11 socket / 设备并传入 `DISPLAY`，否则同样无窗口。 |

启动后请看终端里 **`mujoco_sim_node`** 的日志：若出现 **「未设置 DISPLAY」** 或 **「无法启动 MuJoCo passive viewer」**，即属于上述环境问题，而不是 launch 参数写错。

**临时验证仿真是否在跑（不需要窗口）：**

```bash
ros2 topic hz /go2/joint_states
```

若频率正常（约 200 Hz），说明仿真在运行，仅图形未显示。

**仅用于自动化测试的假显示器（画面体验一般）：**

```bash
sudo apt install -y xvfb
xvfb-run -a ros2 launch go2_rl_control sim.launch.py visualize:=true
```

### 六点六、`mujoco_sim_node` 立刻退出：`ImportError` / `GLIBCXX_3.4.30 not found`

终端里若出现 **`rclpy/_rclpy_pybind11`** 与 **`libstdc++.so.6`** 字样，几乎都是因为：**在 Conda 环境（如 `env_isaaclab`）已激活时运行了 `ros2 launch`**，Python 仍链接到 Conda 自带的旧 `libstdc++`，与 ROS 2 Humble 预编译扩展不兼容。

**处理：**

1. **`conda deactivate`**，确认提示符里没有 `(env_isaaclab)`。
2. 重新 **`source /opt/ros/humble/setup.bash`** 与 **`source ~/ros2_ws/install/setup.bash`**。
3. 本包脚本 shebang 已固定为 **`#!/usr/bin/python3`**；若你仍在使用旧安装，请 **`colcon build --symlink-install --packages-select go2_rl_control`** 后再 launch。

IsaacLab 训练与 ROS 2 仿真建议分两个终端：**训练终端 `conda activate`**，**仿真终端保持系统 Python**。

## 七、在「另一台电脑」上看效果

部署场景分两类：**只看话题数据** 与 **要看 MuJoCo 图形窗口**。

### 7.1 同一局域网：ROS 2 跨机通信（推荐先看文档再试）

1. **两台机器使用相同 `ROS_DOMAIN_ID`（0–232）：**

   ```bash
   export ROS_DOMAIN_ID=42   # 两台设成同一个数字
   ```

2. **关闭或放行防火墙中对 DDS 的 UDP（常见需放行宽范围，按所用中间件文档调整）。**

3. **验证：** 训练电脑上跑第六节栈；另一台同样 `source` ROS 与工作空间后：

   ```bash
   ros2 topic list
   ros2 topic echo /go2/base_pose
   ```

若无法发现话题，多为 **多网卡 / Docker / 防火墙**；可查阅 ROS 2 Humble 官方「跨主机通信」与 DDS 配置（如 CycloneDDS 的 `CYCLONEDDS_URI`）。

### 7.2 要看仿真画面（MuJoCo 窗口）

MuJoCo 可视化必须在 **运行 `mujoco_sim_node` 的那台机器** 上显示窗口，可选方式：

| 方式 | 说明 |
|------|------|
| SSH + X11 转发 | `ssh -X user@训练机IP`，再在该会话里运行 launch（延迟较大，仅适合演示） |
| VNC / NoMachine / 远程桌面 | 在训练机上直接看到桌面与 MuJoCo 窗口 |
| 本地运行 ROS 2 | 仅把 `policy.onnx` 与 `deploy.yaml` 拷到另一台已编译好 `go2_rl_control` 的机器，本地起仿真（需相同依赖） |

**IsaacSim 里用键盘玩：** 在训练/仿真电脑上用 `unitree_rl_lab` 的 `play_keyboard.py`（需 IsaacSim 环境），与 ROS 2 栈相互独立。

在训练用的 shell 里**另开一个终端**执行（会阻塞直到训练进程结束，再刷新并提示启动命令）：

## 九、常见问题

| 现象 | 处理 |
|------|------|
| **`visualize:=true` 无 MuJoCo 窗口** | 见 **第六节「六点五」**；多为无 `DISPLAY`（SSH/云端）。用桌面终端、`ssh -X`、VNC 或 `DISPLAY=:0` |
| `rclpy` 报 `GLIBCXX_3.4.30 not found` | **先 `conda deactivate`**；脚本 shebang 已固定 `/usr/bin/python3`。不要在激活 conda 的终端里运行 `ros2 launch`（除非确认 PATH 不影响） |
| `import mujoco` 失败 | `pip3 install --user mujoco`，并与 `ros2 run` 使用同一 Python |
| 机器人不动 | 确认 `keyboard_teleop` 已发 `/go2/mode` 为 `walk`，且 `/go2/cmd_torque` 有 50 Hz |
| 策略行为差 | 换更大 `model_XXXX.pt` 或等训练完成后再 `refresh_policy.sh` |

## 十一、训练结束后无人值守：等待进程并导出策略

训练若在后台运行，可在训练机上执行（训练结束会自动导出 ONNX 并 `colcon build`）：

```bash
chmod +x ~/ros2_ws/src/go2_rl_control/scripts/wait_train_then_refresh.sh
nohup ~/ros2_ws/src/go2_rl_control/scripts/wait_train_then_refresh.sh >> /tmp/wait_train.log 2>&1 &
tail -f /tmp/wait_train.log
```

可选：导出完成后**自动启动**仿真栈（无窗口，便于远程只订阅话题）：

```bash
RUN_LAUNCH=1 RUN_VISUALIZE=false nohup ~/ros2_ws/src/go2_rl_control/scripts/wait_train_then_refresh.sh >> /tmp/wait_train.log 2>&1 &
```

---

*训练剩余时间请以本机为准：`grep ETA /tmp/go2_train.log | tail -1`（若日志路径不同请改成你的日志文件）。文档内不写固定 ETA，避免过期误导。*
